# Non-goals & Guardrails

## 절대 하지 않음 (MVP)
- OpenClaw 에이전트를 통해 OpenCode를 호출하는 체인
- LLM 이중 호출 구조 (토큰 이중 지출 가능성)
- OpenClaw 내부 자동화 생태계 결합
- 시간/크기 기반 스트리밍 버퍼링 (현재는 이벤트 기반)

## 반드시 지킬 것
- Telegram 입력은 곧바로 로컬 `opencode run --format json`으로 전달
- 출력은 Markdown→HTML 최소 변환만 수행 (구조 보존 우선)
- 실패하면 간단히 우회 (HTML 실패 → plain text 재시도)
- 단일 작업 동시성 보장 (인스턴스당 1개 작업 lock)

## 성공 기준
- 모바일에서 프롬프트 1개를 보내고, 결과 로그를 수신한다.
- 불편해도 된다. 끊겨도 된다. 하지만 "쓸 수 있어야" 한다.

## 현재 달성 상태
- **달성**: Telegram에서 프롬프트 전송 → opencode 실행 → 결과 수신 가능
- **달성**: verbose 모드로 진행 상황 실시간 확인 가능
- **달성**: Markdown 변환으로 가독성 확보
- **달성**: 인스턴스별 독립 실행 + 로그 기록
